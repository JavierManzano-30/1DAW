Funciones en SQL: Guía Teórica y Práctica

En SQL, una función es un bloque de código que realiza una tarea específica y retorna un valor. Hay varios tipos de funciones, pero comúnmente se dividen en:

  • Funciones escalares (devuelven un único valor por fila).
  • Funciones de agregado (efectúan cálculos sobre un conjunto de filas y devuelven un único resultado).
  • Funciones de ventana (similares a las de agregado, pero no “colapsan” las filas en un solo valor; permiten operaciones por “ventanas” o particiones de filas).

Además, existen Funciones definidas por el usuario (UDF), que pueden ser Escalares o de Tabla, según retornen un solo valor o un conjunto de resultados tipo tabla. A continuación, se presentan ejemplos de las funciones más comunes en SQL, cada una con su propia sección desplegable para:

  • Descripción y código
  • Resultado de ejemplo (en un segundo desplegable)

1. Funciones de Agregado (Aggregate Functions)

Se aplican sobre un conjunto de filas y devuelven un solo valor. Suelen usarse con GROUP BY y HAVING.

Descripción y código:

-- Retorna el número de filas que coinciden con la condición especificada
SELECT COUNT(*) AS total_filas
FROM tabla_ejemplo;

/*
Posible resultado:
+--------------+
| total_filas  |
+--------------+
| 12           |
+--------------+
*/

Descripción y código:

-- Devuelve la suma de los valores de una columna numérica
SELECT SUM(precio) AS suma_precios
FROM productos;

/*
Posible resultado:
+---------------+
| suma_precios  |
+---------------+
| 2530.5        |
+---------------+
*/

Descripción y código:

-- Calcula el promedio (media) de los valores de una columna numérica
SELECT AVG(edad) AS promedio_edad
FROM personas;

/*
Posible resultado:
+------------------+
| promedio_edad    |
+------------------+
| 32.4             |
+------------------+
*/

Descripción y código:

-- Devuelve el valor mínimo de una columna
SELECT MIN(salario) AS salario_minimo
FROM empleados;

/*
Posible resultado:
+-----------------+
| salario_minimo  |
+-----------------+
| 1500.00         |
+-----------------+
*/

Descripción y código:

-- Devuelve el valor máximo de una columna
SELECT MAX(salario) AS salario_maximo
FROM empleados;

/*
Posible resultado:
+-----------------+
| salario_maximo  |
+-----------------+
| 5000.00         |
+-----------------+
*/

Ejemplo adicional con GROUP BY y HAVING:

SELECT departamento,
       COUNT(*) AS total_empleados,
       AVG(salario) AS salario_promedio
FROM empleados
GROUP BY departamento
HAVING AVG(salario) > 3000;

/*
Posible resultado (dependiendo de los datos):
+---------------+------------------+-------------------+
| departamento  | total_empleados | salario_promedio  |
+---------------+------------------+-------------------+
| Contabilidad  | 2                | 2850.00           |
| Ventas        | 2                | 3400.00           |
+---------------+------------------+-------------------+
*/

2. Funciones Numéricas

Operan sobre columnas numéricas y devuelven un valor por cada fila.

Descripción y código:

-- Devuelve el valor absoluto de x
SELECT ABS(-15) AS absoluto;  -- Resultado: 15

/*
+----------+
| absoluto |
+----------+
| 15       |
+----------+
*/

Descripción y código:

-- Redondea x hacia arriba al entero más cercano
SELECT CEILING(4.3) AS techo;  -- Resultado: 5

/*
+-------+
| techo |
+-------+
| 5     |
+-------+
*/

Descripción y código:

-- Redondea x hacia abajo al entero más cercano
SELECT FLOOR(4.9) AS piso; -- Resultado: 4

/*
+------+
| piso |
+------+
| 4    |
+------+
*/

Descripción y código:

-- Redondea x al número de decimales especificado
SELECT ROUND(123.4567, 2) AS redondeado; 
-- Resultado: 123.46

/*
+--------------+
| redondeado   |
+--------------+
| 123.46       |
+--------------+
*/

Descripción y código:

-- Retorna x elevado a la potencia y
SELECT POWER(2, 3) AS potencia; -- Resultado: 8

/*
+----------+
| potencia |
+----------+
| 8        |
+----------+
*/

Descripción y código:

-- Devuelve la raíz cuadrada de x
SELECT SQRT(16) AS raiz; -- Resultado: 4

/*
+------+
| raiz |
+------+
| 4    |
+------+
*/

3. Funciones de Cadena

Permiten manipular texto y transformarlo.

Descripción y código:

-- Convierte la cadena a minúsculas
SELECT LOWER('HOLA MUNDO') AS minusculas;

/*
+-------------+
| minusculas  |
+-------------+
| hola mundo  |
+-------------+
*/

Descripción y código:

-- Convierte la cadena a mayúsculas
SELECT UPPER('hola mundo') AS mayusculas;

/*
+-------------+
| mayusculas  |
+-------------+
| HOLA MUNDO  |
+-------------+
*/

Descripción y código:

-- Retorna una parte de la cadena
SELECT SUBSTRING('Hola Mundo', 1, 4) AS sub_cadena; 

/*
+------------+
| sub_cadena |
+------------+
| Hola       |
+------------+
*/

Descripción y código:

-- Devuelve los primeros (LEFT) o últimos (RIGHT) n caracteres
SELECT LEFT('Hola Mundo', 4) AS primeros4; 
SELECT RIGHT('Hola Mundo', 5) AS ultimos5;

/*
Primer query:
+-------------+
| primeros4   |
+-------------+
| Hola        |
+-------------+

Segundo query:
+------------+
| ultimos5   |
+------------+
| Mundo      |
+------------+
*/

Descripción y código:

-- Elimina espacios en blanco al principio (LTRIM) o al final (RTRIM)
SELECT LTRIM('   Hola') AS sin_espacios_inicio;
SELECT RTRIM('Hola   ') AS sin_espacios_final;

/*
LTRIM:
+----------------------+
| sin_espacios_inicio |
+----------------------+
| Hola                |
+----------------------+

RTRIM:
+---------------------+
| sin_espacios_final |
+---------------------+
| Hola               |
+---------------------+
*/

Descripción y código:

-- Reemplaza todas las ocurrencias de 'buscar' en la cadena
SELECT REPLACE('Hola Mundo', 'Mundo', 'SQL') AS nuevo_texto;

/*
+-------------+
| nuevo_texto |
+-------------+
| Hola SQL    |
+-------------+
*/

Descripción y código:

-- Concatena varias cadenas en una sola
SELECT CONCAT('Hola', ' ', 'Mundo') AS cadena_unida;

/*
+---------------+
| cadena_unida  |
+---------------+
| Hola Mundo    |
+---------------+
*/

4. Funciones de Fecha y Hora

Trabajan con valores de tiempo (fecha/hora) y permiten cálculos o conversiones.

Descripción y código:

-- Retorna la fecha y hora actual del servidor (varía según el SGBD)

-- MySQL
SELECT NOW() AS fecha_actual;

-- SQL Server
SELECT GETDATE() AS fecha_actual;

-- Estándar SQL
SELECT CURRENT_TIMESTAMP;

/*
Posible resultado (ejemplo):
+---------------------+
| fecha_actual        |
+---------------------+
| 2025-03-20 10:15:42 |
+---------------------+
*/

Descripción y código:

-- Retorna la fecha actual (sin hora)

SELECT CURDATE() AS solo_fecha;   -- MySQL
SELECT CURRENT_DATE;              -- Estándar

/*
Posible resultado:
+------------+
| solo_fecha |
+------------+
| 2025-03-20 |
+------------+
*/

Descripción y código:

-- Añade un intervalo de tiempo a una fecha dada (varía según el SGBD)

-- SQL Server:
SELECT DATEADD(DAY, 5, '2025-01-01') AS nueva_fecha;

-- MySQL:
SELECT DATE_ADD('2025-01-01', INTERVAL 5 DAY) AS nueva_fecha;

/*
Posible resultado:
+-------------+
| nueva_fecha |
+-------------+
| 2025-01-06  |
+-------------+
*/

Descripción y código:

-- Devuelve la diferencia entre dos fechas en días

-- MySQL:
SELECT DATEDIFF('2025-01-10', '2025-01-01') AS diferencia_dias;

-- SQL Server:
SELECT DATEDIFF(DAY, '2025-01-01', '2025-01-10') AS diferencia_dias;

/*
Posible resultado:
+------------------+
| diferencia_dias  |
+------------------+
| 9                |
+------------------+
*/

Descripción y código:

-- Extrae una parte específica de la fecha (año, mes, día, etc.)

-- Estándar SQL (PostgreSQL, Oracle):
SELECT EXTRACT(YEAR FROM CURRENT_DATE) AS anio;

-- SQL Server:
SELECT DATEPART(YEAR, GETDATE()) AS anio;

/*
Posible resultado:
+------+
| anio |
+------+
| 2025 |
+------+
*/

5. Funciones Lógicas y de Control de Flujo

Expresiones para manejar condiciones y valores NULL.

Descripción y código:

-- Expresión condicional muy usada

SELECT 
    nombre,
    salario,
    CASE 
        WHEN salario < 2000 THEN 'Bajo'
        WHEN salario BETWEEN 2000 AND 5000 THEN 'Medio'
        ELSE 'Alto'
    END AS categoria_salarial
FROM empleados;

/*
Posible resultado (ejemplo):
+--------+---------+--------------------+
| nombre | salario | categoria_salarial|
+--------+---------+--------------------+
| Ana    | 2500.00 | Medio             |
| Juan   | 1800.00 | Bajo              |
| Pedro  | 3200.00 | Medio             |
| Maria  | 5000.00 | Medio (o Alto)*   |
| Lucia  | 4000.00 | Medio             |
+--------+---------+--------------------+
*/

Descripción y código:

-- Retorna el primer valor no nulo de la lista
SELECT COALESCE(telefono_movil, telefono_casa, 'No disponible') AS telefono
FROM contactos;

/*
Posible resultado:
+---------------+
| telefono      |
+---------------+
| 555-1234      |
+---------------+
(O si ambos son NULL, "No disponible")
*/

Descripción y código:

-- Retorna NULL si exp1 = exp2; de lo contrario, exp1
SELECT NULLIF(columna, 0) AS valor
FROM tabla_ejemplo;

/*
Si columna = 0, se mostrará NULL.
De lo contrario, se mostrará el valor de columna.
Posible resultado:
+-------+
| valor |
+-------+
| NULL  |
+-------+
*/

6. Funciones de Ventana (Window Functions)

Permiten realizar operaciones similares a las de agregado, pero sin colapsar los resultados en una sola fila. Se definen con la cláusula OVER y se pueden particionar (PARTITION BY) y ordenar (ORDER BY) de forma independiente.

Ejemplo y posible resultado:

SELECT 
    empleado_id,
    departamento,
    salario,
    ROW_NUMBER() OVER (PARTITION BY departamento ORDER BY salario DESC) AS num_fila
FROM empleados;

/*
Posible resultado:
+-------------+---------------+---------+----------+
| empleado_id | departamento  | salario | num_fila |
+-------------+---------------+---------+----------+
|     3       | Contabilidad  | 3200.00 |    1     |
|     1       | Contabilidad  | 2500.00 |    2     |
|     4       | Ventas        | 5000.00 |    1     |
|     2       | Ventas        | 1800.00 |    2     |
|     5       | IT            | 4000.00 |    1     |
+-------------+---------------+---------+----------+
*/

7. Funciones Definidas por el Usuario (UDF)

La sintaxis varía según el motor de base de datos. Un ejemplo en SQL Server: 

CREATE FUNCTION fn_CalcularIVA
(
    @precio DECIMAL(10,2), 
    @tasa DECIMAL(5,2)
)
RETURNS DECIMAL(10,2)
AS
BEGIN
    DECLARE @valorIVA DECIMAL(10,2);
    SET @valorIVA = @precio * @tasa;
    RETURN @valorIVA;
END;

SELECT dbo.fn_CalcularIVA(100, 0.21) AS iva_calculado; 
-- Resultado: 21.00

En MySQL:

DELIMITER $$
CREATE FUNCTION fn_calcularIVA(precio DECIMAL(10,2), tasa DECIMAL(5,2))
RETURNS DECIMAL(10,2)
DETERMINISTIC
BEGIN
    DECLARE valorIVA DECIMAL(10,2);
    SET valorIVA = precio * tasa;
    RETURN valorIVA;
END $$
DELIMITER ;

SELECT fn_calcularIVA(100, 0.21) AS iva_calculado;
/*
Posible resultado:
+---------------+
| iva_calculado |
+---------------+
| 21.00         |
+---------------+
*/

8. Ejemplos Prácticos

Sentencias que puedes ejecutar (adaptar a tu SGBD). Primero, creamos una base de datos y tabla de ejemplo: 

-- Crear base de datos (opcional, depende de tu SGBD)
CREATE DATABASE mi_basedatos;
USE mi_basedatos;  -- MySQL

-- Crear tabla de ejemplo
CREATE TABLE empleados (
    empleado_id INT PRIMARY KEY,
    nombre VARCHAR(50),
    departamento VARCHAR(50),
    salario DECIMAL(10,2),
    fecha_ingreso DATE
);

-- Insertar datos de prueba
INSERT INTO empleados (empleado_id, nombre, departamento, salario, fecha_ingreso)
VALUES
(1, 'Ana', 'Contabilidad', 2500.00, '2022-01-15'),
(2, 'Juan', 'Ventas',       1800.00, '2021-11-10'),
(3, 'Pedro', 'Contabilidad', 3200.00, '2019-06-01'),
(4, 'Maria', 'Ventas',       5000.00, '2020-02-20'),
(5, 'Lucia', 'IT',           4000.00, '2021-08-30');

Consultas de ejemplo (con posibles resultados):

SELECT COUNT(*) AS total_empleados,

9. Consejos Generales

  • Verifica la compatibilidad: Algunas funciones varían entre distintos SGBD.
  • Usa el manual oficial: Para confirmar sintaxis y disponibilidad.
  • Optimiza: Evita usar funciones en WHERE si la columna está indexada.
  • Organiza tu código: Separa secciones (creación de tablas, inserción de datos, etc.).
  • Ejemplos con datos reales: Facilita la comprensión en trabajos académicos o presentaciones.

Conclusión

En SQL, las funciones son herramientas poderosísimas para cálculos, formateo de datos y transformaciones. Conocer tanto las funciones de agregado como las escalares (numéricas, cadena, fecha/hora), las funciones de ventana y las UDF te permitirá crear consultas avanzadas de manera eficiente.
